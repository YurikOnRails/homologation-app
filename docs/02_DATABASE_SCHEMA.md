## Tables Overview

| Group | Table | Color | Purpose |
|---|---|---|---|
| Auth | `users` | 🔵 | All users (students, teachers, coordinators, admins) |
| Auth | `roles` | 🔵 | 4 roles: super_admin, coordinator, teacher, student |
| Auth | `user_roles` | 🔵 | Join table (user can have multiple roles) |
| Auth | `sessions` | 🔵 | Rails 8 auth sessions |
| Teachers | `teacher_profiles` | 🟠 | Level, hourly rate, permanent meeting link |
| Teachers | `teacher_students` | 🟠 | Many-to-many, assigned by coordinator |
| Teachers | `lessons` | 🟠 | Calendar: scheduled lessons with meeting links |
| Business | `homologation_requests` | 🔴 | Core: student requests for document homologation |
| Chat | `conversations` | 🟢 | Request chats + teacher-student chats |
| Chat | `messages` | 🟢 | Chat messages (real-time via Action Cable) |
| Notifications | `notifications` | 🟡 | In-app notifications |
| CRM | `amo_crm_tokens` | 🟣 | AmoCRM OAuth2 tokens |

Total: **12 tables** (+ Active Storage auto-generated by Rails)

|---|---|---|
| HomologationRequest | `application` | has_one_attached |

| HomologationRequest | `documents` | has_many_attached |

Max 10 MB per file. No S3 — local disk only.

## Key Relationships

- `users` ←many-to-many→ `roles` (via `user_roles`)
- `users` ←one-to-one→ `teacher_profiles` (only for teachers)
- `teachers` ←many-to-many→ `students` (via `teacher_students`, assigned by coordinator)
- `teacher + student` → `lessons` (calendar entries)
- `homologation_request` ←one-to-one→ `conversation` (request chat)
- `teacher_student` ←one-to-one→ `conversation` (teacher-student chat)
- `conversation` → `messages` (chat history)
