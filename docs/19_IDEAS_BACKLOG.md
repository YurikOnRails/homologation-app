# Ideas Backlog

> Обсудить с admin перед реализацией.

## 1. Прогресс-бар для студента

Студент видит только статус "In Progress", но admin двигает карточку по 8 стадиям pipeline. Показать студенту визуальный прогресс ("Этап 4/7: Tasas y Volantes") — сократит вопросы "ну что там?" в чате.

**Эффект:** экономия времени admin'а на повторяющиеся вопросы.

## 2. Уведомления при смене стадии pipeline

`advance_pipeline!` меняет стадию, но студент узнаёт только если зайдёт в приложение. Автоматическое уведомление (in-app + email + Telegram) при каждом advance/retreat.

**Эффект:** студент в курсе без лишних вопросов.

## 3. "Предпайплайн" — заявки ожидающие оплату

Компактная полоса над канбаном с заявками в `awaiting_payment` — имя, сумма, дата — с кнопкой "напомнить" (отправить уведомление студенту).

**Эффект:** ускоряет конверсию, полная картина для admin.

## 4. Убрать AmoCRM-интеграцию

Приложение уже покрывает весь workflow: pipeline, чат, документы, статусы. AmoCRM дублирует данные и добавляет сложность (sync jobs, API errors). Обсудить нужна ли она ещё.

**Эффект:** упрощение кодовой базы, меньше багов.
